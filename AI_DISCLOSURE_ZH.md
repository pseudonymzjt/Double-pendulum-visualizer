# AI 公开声明 — 双摆可视化器

> **语言**：中文 · [English](AI_DISCLOSURE.md)

## 项目概述

**混沌艺术 — 双摆可视化器** 是一个基于 HTML5 Canvas 的 Web 应用，能够模拟 N 摆混沌系统并实时展示相空间分析。本项目由人类开发者与 AI 编程助手在多个开发阶段中协同完成。

## 使用的 AI 工具

| 工具 | 角色 | 范围 |
|------|------|------|
| **Claude (Anthropic)** | 主要 AI 编程助手 | 全部开发阶段 — 架构设计、代码实现、调试、文档 |
| **Claude Code** | AI 驱动的 CLI/IDE 编码代理 | 代码生成、重构、缺陷分析、文件编辑、Git 操作 |

本项目未使用其他 AI 工具（如 ChatGPT、Copilot、Gemini 等）。

## AI 的贡献

### 代码生成

AI 助手生成了全部三个文件（`index.html`、`style.css`、`script.js`）中的绝大部分源代码，包括：

- **HTML 结构**：极简外壳，包含控制栏、参数滑块、分析面板、上下文菜单和双画布布局。
- **CSS 样式**：全暗色主题（`#0a0a0f`）、毛玻璃 UI 元素、响应式媒体查询、触控加固（`touch-action: none`、`overscroll-behavior: none`）。
- **JavaScript 物理引擎**：基于拉格朗日力学的 RK4 积分、N×N 质量矩阵的高斯消元求解器、能量计算、Verlet 积分（最初使用，后被替换）。
- **渲染系统**：双层 HiDPI 画布架构、批处理渐隐轨迹、基于速度的动态线宽、霓虹辉光效果。
- **分析图表**：使用原生 Canvas 2D 绘制的相空间图和能量时序图（无外部库依赖）。
- **UI 交互**：拖拽设置、角度吸附、多摆选择、上下文菜单、参数滑块、点击放大、键盘快捷键。

### 架构决策

AI 根据用户反馈提出并实施了多项重要的架构变更：

| 阶段 | 决策 | AI 角色 |
|------|------|---------|
| 第1–2阶段 | 双层画布 + 批处理渐隐轨迹 | 架构提议与实现 |
| 第3阶段 | 基于数组的多摆状态模型（混沌模式） | 架构提议 |
| 第7阶段 | 使用 Verlet 积分实现 N 节链 | 实现（后被发现有数值阻尼问题） |
| 第8阶段 | 用拉格朗日 RK4 替换 Verlet（精确物理） | 阻尼根因分析，提出并实现 N×N 质量矩阵求解器 |
| 第8阶段 | 每个摆独立维护指标缓冲区 + `globalMetricsStep` | 调试能量图 X 轴停滞问题 |
| 第9阶段 | 触控加固、响应式 CSS、可折叠设置面板 | 实现 |

### 调试与缺陷修复

AI 分析并修复了多个缺陷，详见 `THOUGHTS.md`：

1. **相图跨图连线** — 检测到角度回绕导致直线穿越图表；通过在 `drawFadingLine` 中设置回绕阈值修复。
2. **能量图 X 轴停滞** — 诊断出环形缓冲区索引复用问题；通过 `globalMetricsStep` 计数器修复。
3. **总能漂移** — 识别出势能公式中符号错误（y轴向上与向下约定的差异）；修复后漂移量降至 0.0000%。
4. **启动时画布空白** — 发现缺少同步初始绘制和不安全的 `matchMedia` 调用；两者均已修复。
5. **GitHub Pages 空指针崩溃** — 诊断出缺少 `defer` 的脚本执行时机问题及缺失的 DOM 元素；通过 `$()`/`on()` 辅助函数和 18 处空安全事件绑定修复。
6. **Emoji 渲染为豆腐块** — 将系统 emoji 替换为内联 SVG，实现跨平台兼容。
7. **角度显示点击无响应** — 识别出 `stopPropagation` 阻塞事件传递；改用 `pointerdown` 事件委托。
8. **右键点击隐藏上下文菜单** — 添加 `e.button !== 0` 守卫条件。

### 质量控制

人类拒绝了不满意的输出并要求重做：

| 被拒绝的内容 | 后续改进 |
|-------------|----------|
| *"双摆的轨迹看起来太规律了，缺乏混沌运动"* | 将 Verlet 积分替换为拉格朗日 RK4 |
| *"所有的图看起来都和动画一样"* | 增加了具有不同观察角度的专业相图 |
| *"图上的数值和坐标轴太暗了"* | 全面提亮（网格 0.06→0.14，轴线 0.12→0.35） |
| *"最后两个摆锤完全锁定在一起"* | 将 2×2 求解器推广为 N×N 质量矩阵 |
| *"你把第二个图上的字母搞得太大了，反而看不清"* | 将粗体大字图例替换为紧凑的彩色圆点图例 |
| *"移除 0.5 倍速选项，它已经集成到滑条里了"* | 移除冗余的慢动作按钮 |
| *"图表的坐标轴字母不需要下标"* | 简化坐标轴标签 |

## 协作工作流

```
1. 人类：  提出需求或描述期望的功能
2. AI：    提议实现方案（可能包含代码和架构选项）
3. 人类：  审查方案，批准或要求调整方向
4. AI：    编写/修改代码，更新文档
5. 人类：  在浏览器中测试结果（笔记本和/或手机）
6. 人类：  报告缺陷或请求优化
7. AI：    调试，提出修复方案，实施修复
8. 人类：  审查修复，批准提交
   → 回到第1步，进入下一功能迭代
```

所有提交均由人类发起并批准。AI 在未获得人类明确授权的情况下不会推送代码。

## 局限性与透明度声明

### 已知约束

- **AI 并未实际运行应用程序**：所有测试均由人类执行。AI 仅通过静态代码分析和错误报告进行调试。
- **AI 的物理知识为辅助性质**：拉格朗日力学的实现是通过人类对能量守恒的观察（总能线保持水平）来验证的，而非通过形式化证明。
- **Verlet 积分的技术绕路**：AI 最初按计划实现了 Verlet 积分（第7阶段）。人类发现其产生过于规则的运动后，AI 诊断出根因为数值阻尼，并提出了拉格朗日 RK4 替换方案——这一修正得益于人类的领域观察能力。

### 归属说明

- 代码风格、注释规范和项目结构反映了 AI 的默认模式，随后根据人类的偏好逐步调整（例如"少写注释，代码自文档化"）。
- 物理公式（拉格朗日量、质量矩阵、高斯消元法）均为标准教科书内容——AI 根据数学描述将其转写为可运行代码。
- 所有内联 SVG 图标均由 AI 在人类报告 emoji 渲染失败后生成。

## 结语

本项目是真正的人机协作成果：人类提供了领域知识、方向指引、测试验证和质量把关；AI 提供了代码生成、快速实现、调试分析和文档撰写。每一项设计决策都源于人类的需求，每一行代码输出在纳入项目前都经过人类的审查和认可。
